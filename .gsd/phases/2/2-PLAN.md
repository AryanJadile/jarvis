# Plan 2.2: Android Text-to-Speech (TTS)

<objective>
Implement Android's built-in `TextToSpeech` engine to read aloud the `spokenResponse` generated by Gemini, giving Jarvis a voice.

Purpose: To provide audible feedback so the user doesn't have to look at the screen to know Jarvis understood the command.
Output: Audible speech playing from the device speakers (e.g., "I've set your alarm for 7 AM.").
</objective>

<context>
Load for context:
- .gsd/SPEC.md
- app/src/main/java/com/jarvis/JarvisService.kt
</context>

<tasks>

<task type="auto">
  <name>Initialize TextToSpeech</name>
  <files>app/src/main/java/com/jarvis/JarvisService.kt</files>
  <action>
    Add a `TextToSpeech` instance to `JarvisService`.
    Implement `TextToSpeech.OnInitListener` on the Service.
    In `onCreate`, instantiate `tts = TextToSpeech(this, this)`.
    In `onInit`, check if `status == TextToSpeech.SUCCESS`. If so, set the language to `Locale.US` or the device default.
    AVOID: DO NOT call `speak` immediately in `onCreate`. The engine takes time to initialize asynchronously.
  </action>
  <verify>cat app/src/main/java/com/jarvis/JarvisService.kt | Select-String "TextToSpeech"</verify>
  <done>TextToSpeech object is created and initialized successfully within the Service lifecycle.</done>
</task>

<task type="auto">
  <name>Speak Gemini Response</name>
  <files>app/src/main/java/com/jarvis/JarvisService.kt</files>
  <action>
    Locate the coroutine where Gemini's JSON response is received.
    Extract the `spokenResponse` string from the JSON string. You can use standard Kotlin `JSONObject(responseString).optString("spokenResponse")`.
    Call `tts?.speak(spokenResponse, TextToSpeech.QUEUE_FLUSH, null, "JarvisUtteranceId")`.
  </action>
  <verify>Review `JarvisService.kt` to ensure `tts?.speak` is called after a successful Gemini generation.</verify>
  <done>Jarvis audibly speaks the generated acknowledgment string.</done>
</task>

<task type="auto">
  <name>Clean Up Resources</name>
  <files>app/src/main/java/com/jarvis/JarvisService.kt</files>
  <action>
    In `onDestroy()`, ensure `tts?.stop()` and `tts?.shutdown()` are called to prevent memory leaks.
  </action>
  <verify>cat app/src/main/java/com/jarvis/JarvisService.kt | Select-String "tts\?\.shutdown"</verify>
  <done>TTS engine is properly shut down when the service dies.</done>
