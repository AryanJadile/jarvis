# Phase 2 Research: Gemini LLM Integration

## Objective
Determine the best way to integrate the Gemini LLM into our native Android app to parse transcribed voice commands into structured, executable intents.

## Integration Options

### 1. Google AI client SDK for Android
- **Pros:** Official, idiomatic Kotlin API provided by Google. Handles REST calls, serialization, and streaming out of the box. Easy to use `GenerativeModel`.
- **Cons:** Requires adding the `com.google.ai.client.generativeai` dependency. Relies on the user bringing their own API key (which is fine for this project).
- **Decision:** **Selected.** It is the fastest and most reliable way to integrate Gemini into a native Android app without building custom Retrofit clients.

### 2. Custom Retrofit/OkHttp REST Client
- **Pros:** Full control over the network stack. Less dependencies.
- **Cons:** Reinventing the wheel. Manual JSON serialization/deserialization of complex Gemini request/response objects.
- **Decision:** Rejected.

## Prompt Engineering Strategy (Structured Output)

To make Jarvis ACTUALLY do things (instead of just chatting), we need Gemini to act as a parser, not a conversational agent.

When `JarvisService` gets the transcribed text (e.g., "Jarvis, tell mom I'll be 5 minutes late"), it needs to know *what Android Intent* to fire.

We will use Gemini's **System Instructions** and request a **JSON Response Format**.

### The Prompt Template
**System Instruction:**
"You are the brain of an Android mobile assistant. Your job is to parse the user's spoken command and determine which of the 4 supported intents they want to execute:
1. SEND_WHATSAPP (Requires 'targetName' and 'messageText')
2. SET_ALARM (Requires 'timeHour' and 'timeMinute' in 24h format)
3. MAKE_CALL (Requires 'targetName')
4. TAKE_NOTE (Requires 'noteContent')
5. UNKNOWN (If the command doesn't match the above)

You must ALWAYS respond in valid JSON matching this schema:
{
  \"intent\": \"<INTENT_NAME>\",
  \"parameters\": { ... },
  \"spokenResponse\": \"<A short, natural sentence to speak back to the user acknowledging the action>\"
}"

**User Input:** The transcribed text string from the Android `SpeechRecognizer`.

## Audio Feedback (TTS) Options

### 1. Android `TextToSpeech` (Built-in)
- **Pros:** Free, built-in, completely offline. Can speak the `spokenResponse` generated by Gemini.
- **Decision:** **Selected.**

## Required Setup
- Add `implementation("com.google.ai.client.generativeai:generativeai:0.2.2")` to `build.gradle.kts`.
- Add a new config field for `GEMINI_API_KEY` in `local.properties`.
- Create a handler class (e.g., `BrainManager`) inside `JarvisService` to encapsulate the LLM calls and TTS logic.
